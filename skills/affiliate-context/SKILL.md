## 4. NG（やらないこと）

- 体験していない商品を体験談として書かない
- 効果を断定する誇大表現を使わない（特に健康/美容/金融ジャンル: 薬機法・景表法）
- PR表記・アフィリエイト表記を省略しない

## 5. 必須の表記（法令順守 / コピペ用）

記事には必ず以下を含めます（ステマ規制 2023年10月〜 / 景表法）:

```
※本記事はアフィリエイトプログラムを利用したPR（広告）を含みます。
※Amazonのアソシエイトとして、適格販売により収入を得ています。
```

## 6. 投稿先設定

- 使用プラットフォーム: **note**（手貼り運用）
- 投稿方法: 手動コピペ（noteは公式投稿APIが無いため）
- note向け出力ルール:
  - Markdownの表は使わない（noteエディタが描画しないため、箇条書きに変換）
  - 見出しは `##`（noteの「大見出し」に対応）まで。`###`は太字＋改行で代用
  - アフィリリンクは記事内に **1本だけ**（羅列禁止。noteの規約リスクを下げる）
  - リンクはAmazon等の素のURLを1行で置く（noteがカード展開する）

## 私（Claude）が記事を作るときの手順

ユーザーが「商品URL + 使ってみた感想」を渡したら、以下を順に実行する:

1. このSKILL.mdを読み、運営者プロフィール・ペルソナ・表記を反映する
2. 商品の使用感を `articles/TEMPLATE.md` の構成に沿って記事化する
3. E-E-A-T（実体験の固有名詞・数字）を必ず入れる
4. 第6章のルール（AI臭を消すNGリスト / 同じ語尾3連続禁止 / 冒頭の挨拶禁止）を守る
5. PR表記・アソシエイト表記を必ず入れる
6. `articles/YYYY-MM-DD-<slug>.md` として保存する
7. 投稿先が自動化対応なら `scripts/` の投稿スクリプトで下書き投稿する

# アフィリエイト永続コンテキスト（SKILL.md）

このファイルは「毎回前提を貼り直す」手間をなくすための永続メモリです。
記事生成タスクのたびに、最初にこの内容を読み込んでから作業します。
（出典: じゃがくん記事 第7章 テクニック29 / Filesystem MCP + Claude Skills）

---

## 1. 運営者プロフィール（E-E-A-T の核）

- 発信者の立場 / 肩書き: **20代・メーカーのIT部門勤務**。効率・コスパ重視の視点で選ぶ
- 文体・一人称: です・ます調 / 一人称「私」。誇張しない、正直に書く
- 得意ジャンル: 時短グッズ / 生活便利アイテム / ガジェット
- アカウント名: 使ってよかったモノ図鑑（note ID: katte_yokatta）

## 2. ターゲット読者（ペルソナ）

- 想定読者: 20〜30代、買って後悔したくない／効率よく暮らしたい人
- 読者が抱える悩み: 時間がない、何を買えばいいか迷う、失敗したくない

## 3. 扱う商品リスト（実際に使って良かったもの）

| 商品名 | アフィリエイトURL | 使った期間 | 良かった点 | イマイチな点 |
|---|---|---|---|---|
| （未記入） | | | | |

> 運用ルール: 私（Claude）は **このリストにある商品しか「おすすめ」として書きません**。
> 使っていない商品を実体験のように書くことは絶対にしません（ステマ・景表法違反になるため）。

## 4. NG（やらないこと）

- 体験していない商品を体験談として書かない
- 効果を断定する誇大表現を使わない（特に健康/美容/金融ジャンル: 薬機法・景表法）
- PR表記・アフィリエイト表記を省略しない

## 5. 必須の表記（法令順守 / コピペ用）

記事には必ず以下を含めます（ステマ規制 2023年10月〜 / 景表法）:

```
※本記事はアフィリエイトプログラムを利用したPR（広告）を含みます。
※Amazonのアソシエイトとして、適格販売により収入を得ています。
```

## 6. 投稿先設定

- 使用プラットフォーム: **note**（手貼り運用）
- 投稿方法: 手動コピペ（noteは公式投稿APIが無いため）
- note向け出力ルール:
  - **noteはMarkdown非対応**。チャットに出す「note貼り付け用テキスト」は `##`・`-` を使わず素のテキストにする。
    見出しは見出しテキストだけを行にし、「この行を見出しにする」とユーザーに案内する。
  - 表は使わない（描画されない）。箇条書きは「・」や絵文字で表現する。
  - アフィリリンクは原則 **1本**（規約リスク低減。ただし用途が違う複数オプションは可）。
  - カード化はフルURL（amazon.co.jp/dp/ASIN）。タグを効かせるにはnote側でアソシエイトID登録が前提。
    未登録なら amzn.to のアフィリリンクを素のテキストで貼る（報酬確実）。
  - リポジトリ内の `.md` ファイルは記録用。チャットには毎回「コピペ用テキスト」を別途出す。

## 私（Claude）が記事を作るときの手順

ユーザーが「商品URL + 使ってみた感想」を渡したら、以下を順に実行する:

1. このSKILL.mdを読み、運営者プロフィール・ペルソナ・表記を反映する
2. **`articles/ARTICLE_STRUCTURE.md` の「黄金構成」に必ず従って**記事化する
   （結論ファースト → おすすめする人/しない人 → なぜ買った → 実体験 → 良かった点 → 気になった点 → まとめ）
3. E-E-A-T（実体験の固有名詞・数字・使ったシーン）を必ず入れる
4. 文章ルール（AI臭ワード禁止 / 同じ語尾3連続禁止 / 冒頭の挨拶禁止 / 一文を短く）を守る
5. デメリットを必ず正直に書く（信頼の源）
6. PR表記・アソシエイト表記を必ず入れる
7. `articles/YYYY-MM-DD-<slug>.md` として保存する

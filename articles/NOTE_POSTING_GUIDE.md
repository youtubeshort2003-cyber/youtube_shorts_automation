# noteへの投稿手順（手貼り運用）

noteには記事を自動投稿する公式APIが無いため、最後の「貼り付け」だけ手作業です。
ここを30秒で終わらせるための手順をまとめます。

## 投稿の流れ

1. 私（Claude）が `articles/YYYY-MM-DD-<商品名>.md` に **note向けフォーマット**で記事を生成
2. あなたがそのファイルを開き、本文をコピー
3. note の「記事を書く」→ 本文に貼り付け
4. タイトルはファイル先頭の `title:` を使う
5. アフィリンク（Amazon等のURL）の行はそのまま貼ると、noteが自動でカード表示にする
6. 公開前に「PR表記」が入っているか目視（ここだけは必ず人間がチェック）

## noteエディタのクセ（重要・必読）

- **noteはMarkdownに非対応**。貼り付けた `##` や `- ` は記号のまま表示されてしまう。
  → 記事の「note貼り付け用テキスト」は **`##` や `-` を使わない素のテキスト**で出す。
  → 見出しは、ユーザーがnote上で対象行を選択 →「見出し」ボタンで設定する（手動）。
- **表（テーブル）は使えない** → 箇条書き（「・」や絵文字）に変換して出力する。
- リンクをカードにするには、URLを**空の行に貼ってEnter**。短縮（amzn.to）でカードにならない時は
  **フルURL（https://www.amazon.co.jp/dp/ASIN）** を使う。
- ただしフルURLでカード化する場合、自分のタグを効かせるには
  **note設定でアソシエイトID（kattelog-22）の事前登録が必須**（note.comのブラウザ版で設定）。
  未登録なら、報酬を確実にするため amzn.to のアフィリリンクを素のテキストで貼る。

## アフィリエイト規約リスクを下げるための運用ルール

note は「アフィリエイトのみを目的とした投稿」を制限しています。停止を避けるため:

- 1記事につき **アフィリリンクは1本だけ**（商品比較で複数貼らない）
- 記事の主役は「実体験レビュー」。リンクはおまけの位置づけ
- 記事冒頭に PR表記を必ず入れる（ステマ規制対応 / 私が自動で入れます）
- 同じ商品リンクを量産記事で使い回さない

> もし将来「リンクをたくさん貼りたい / 本格的に稼ぎたい」となったら、
> note ではなく自前の WordPress ブログへの移行を推奨します（アフィリ自由・私がAPI自動投稿可）。
